These pages represent my complete individual contribution to the group website. My focus was on creating an engaging site entry experience, a well-structured interactive simulator, and a clearly documented content page on Life on Land.

Splash Screen

HTML Structure

The splash screen (splash.html) is a standalone full-screen page that acts as the entry point to the website before the user reaches home.html. It displays the website title, a short tagline, and an "Enter Site" button. I used a <div> centred with Flexbox to position the content in the middle of the viewport.

CSS Techniques

I used embedded CSS to create a full-screen background with a dark green overlay, a fade-in animation on the title and button using @keyframes, and a smooth hover effect on the enter button. I used 100vh to make the page fill the full browser height.

Design and Implementation Rationale

The splash screen was designed to make a strong first impression while staying visually consistent with the green environmental theme. I kept the layout minimal so users are immediately drawn to the enter button. A small JavaScript redirect sends the user to home.html when the button is clicked.

Action Simulator Page (AIS)

HTML Structure

The Action Impact Simulator (ais.html) is titled "TerraGuard | SDG 15 Action Impact Simulator". It uses the shared header and footer and adds a page-specific banner followed by a card grid inside <div class="ais-grid">. Each card contains a relevant image, a bold action title, and a short description of the conservation benefit.

CSS Techniques

I created a separate ais.css file for all page-specific styles. Key techniques used include:

  • CSS Grid with grid-template-columns: repeat(3, 1fr) for exactly 3 cards per row
  • A dark nature background image with a semi-transparent rgba overlay to keep text readable
  • Card hover effect using transform: translateY(-5px) and box-shadow
  • Responsive breakpoints — 2 columns on tablet, 1 column on mobile

Design and Implementation Rationale

I chose a 3-column grid so all action cards display clearly without excessive scrolling. The dark blurred forest background creates an immersive environment while the white cards on top remain easy to read. Card titles use dark red (#8B0000) to stand out clearly against the white card background.

Content Page — Life on Land

HTML Structure

My content page (content_ST2.html) covers Life on Land across six sections: Introduction, Importance of Trees, Role of Animals, How Plants Help, Human Action, and Conclusion. I used a two-column layout with a sticky sidebar on the left and the main article content on the right, wrapped inside <div class="s2-page container">. Each section has a unique id so sidebar links can jump directly to it. A "Back to Top" anchor link is provided at the bottom.

CSS Techniques

I created a separate Content.css file with all classes prefixed s2- to avoid any conflict with the shared style.css. Key techniques include:

  • Flexbox for the two-column sidebar and content layout
  • position: sticky on the sidebar so it stays visible while scrolling
  • Green heading bars on each section card using background-color on the h2
  • A pull-quote styled with a coloured left border and light tinted background in the conclusion section
  • Responsive stacking on screens below 768px

Design and Implementation Rationale

I wrote original content for every section rather than copying SDG material, as the coursework requires meaningful original work. I chose a sticky sidebar so users navigating long content always have quick access to other sections. The green heading bar on each card connects visually to the shared global palette.

Validation Page

HTML Structure

The validation page (validation_ST2.html) documents W3C HTML and CSS validation results for all pages I created. It is divided into one section per page, each containing a screenshot of the W3C validator result and a short written note describing any issues found and how they were fixed.

Design and Implementation Rationale

I kept the validation page clear and straightforward so the evidence is easy to read and assess. Each section clearly labels which page is being validated and shows the result alongside a brief explanation. This satisfies the coursework requirement to provide validation evidence for every page I implemented.

Page Editor (This Page)

This Page Editor (pageEditor_ST2.html) documents all of my individual contributions to the group coursework. It uses the same shared HTML structure as all other site pages — global header, navigation, and footer — with style.css for shared styling. All page-specific styles are added in an embedded <style> block in the <head>.

The layout uses CSS Grid with a sticky sidebar contents list on the left and individual documentation sections on the right, following the same card pattern used across the site for visual consistency.

Challenges and Lessons Learned

On the splash screen, the challenge was making the animation feel smooth without slowing page load. I solved this by using only CSS @keyframes with opacity and transform — no heavy JavaScript effects.

For the AIS page, the main challenge was keeping the 3-column grid balanced on different screen sizes. I solved this with responsive CSS Grid breakpoints, reducing to 2 columns on tablet and 1 column on mobile.

On the content page, using the s2- prefix on every CSS class was essential to prevent conflicts with the shared stylesheet. This taught me the importance of clear naming conventions in team projects.

From this project I learned:

  • How to build a full-screen animated entry page with CSS keyframes
  • How to use CSS Grid for clean, responsive multi-column card layouts
  • How to write page-specific CSS alongside a shared global stylesheet without conflicts
  • How to document technical decisions clearly and professionally

Accessibility Considerations

Across all my pages I used semantic HTML elements — <header>, <nav>, <main>, <section>, and <aside> — to help assistive technologies interpret the page structure correctly. All images include meaningful alt text.

On the content page, the sticky sidebar uses an aria-label attribute to identify it as page section navigation. A "Back to Top" link at the bottom of the content area supports keyboard navigation through long pages.

I also applied the following practices across my pages:

  • High contrast between text and background colours throughout
  • Clear heading hierarchy using h1 through h3
  • Focus states inherited from the global CSS on all links and buttons
  • Responsive layouts that work on small screens without horizontal scrolling

Compliance with JANET Regulations

All pages I created contain original, appropriate educational content focused on SDG 15 — Life on Land. I did not include any harmful, misleading, or unlawful material, and I avoided any personal data collection features on my pages.

Images used across my pages are sourced from royalty-free libraries and referenced appropriately. I did not copy written content from external sources without proper attribution. My implementation also supports accessible and inclusive design in line with JANET compliance guidance for educational institutions.

My pages support responsible publishing through original content, accessible design, appropriate educational subject matter, proper use of external assets, and no unnecessary personal data collection.
