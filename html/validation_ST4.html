Reflection on Validation

Validating my pages was a real eye-opener. When I ran my User Profile, Sitemap, and Content pages through the W3C tool, I found a few small mistakes that I hadn't even noticed. For example, on my SVG Sitemap, I forgot to close one of my tags properly and missed adding a title for screen readers. On my Tech Waste content page, I accidentally used the exact same id twice for my navigation links.

Fixing them was pretty easy once the validator pointed them out. I just went back into my code, renamed the duplicate IDs so my anchor links would work properly, and made sure all my SVG tags were closed off correctly.

The biggest lesson I learned from this is that a website can look completely fine in the browser, but still have messy code behind the scenes. Using the validator forced me to write cleaner, standard-compliant HTML that works properly across all devices.
