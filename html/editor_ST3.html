These pages represent my individual contribution to the group website. My focus was on creating interactive, user-friendly, and visually consistent pages that support both content delivery and user engagement.

Feedback Page

HTML Structure

The feedback page (feedback.html) was designed to collect user opinions and suggestions about the website. The page includes a structured feedback form with fields such as name, email, rating options, comments, and submission controls. I used semantic HTML elements and grouped related form controls clearly to improve readability and usability.

CSS Techniques

I used a separate CSS file to style the feedback page with a clean and professional dashboard-like layout. Flexbox and grid techniques were used to align form elements properly. I also styled radio buttons, checkboxes, buttons, and containers to create a neat and responsive design.

Design and Implementation Rationale

The feedback page was created to make user interaction simple and effective. I focused on keeping the layout clear, professional, and easy to use. Proper spacing, alignment, and consistent colour usage were applied to make the form more attractive and user friendly.

Team Page

HTML Structure

The team page (team.html) presents the group members and their assigned roles. The page uses a structured layout with individual member cards containing images, names, roles, and task details. Each member section was organised clearly so users can easily understand the contribution of each student.

CSS Techniques

I used CSS to style the team cards with consistent spacing, hover effects, and alignment. Flexbox was used to arrange the team members in a responsive layout. Styling was added to improve visual balance, readability, and overall presentation.

Design and Implementation Rationale

The purpose of the team page was to introduce the project team in a clear and visually appealing way. I designed it so that each member’s role and assigned work can be viewed easily, while also maintaining consistency with the overall theme of the website.

Content Page

HTML Structure

My content page (content_ST3.html) was developed to present topic-related information in a structured and readable format. The page includes multiple content sections, headings, paragraphs, and internal navigation links to improve the reading experience. Semantic elements such as <section> and <article> were used to organise the content properly.

CSS Techniques

I used CSS to style the content page with a clean layout, readable typography, and consistent spacing. Section styling, background colours, and heading designs were used to separate the content clearly. Responsive styling was also applied so the page works well on different screen sizes.

Design and Implementation Rationale

I designed the content page to make information easy to read and visually engaging. My main goal was to present the content in a well-structured way while maintaining the design consistency of the overall website.

Validation Page

HTML Structure

The validation page (validation_ST3.html) was created to provide validation evidence for all the pages I developed. It includes separate sections for each page, along with screenshots and short explanations of the validation results.

Design and Implementation Rationale

I kept the validation page simple and organised so that the evidence can be reviewed easily. This page helps demonstrate that my HTML and CSS were checked and improved based on validation results, which supports the technical quality of my work.

Page Editor (This Page)

This Page Editor (pageEditor_ST3.html) documents all of my individual contributions to the group coursework. It uses the shared website structure, including the global header, navigation, and footer, while presenting my work in a clear and well-organised format.

The page includes a sidebar navigation menu and multiple content sections so that users can easily move through the documentation. This helps present my contribution in a structured and professional way.

Challenges and Lessons Learned

One of the main challenges I faced was maintaining proper alignment and spacing across different page elements, especially in form components and team member layouts. I solved this by using Flexbox and careful CSS adjustments.

Another challenge was keeping all my pages visually consistent with the shared website theme while also making each page unique in purpose and structure.

From this project I learned:

  • How to design and structure user-friendly web pages
  • How to build responsive layouts using HTML and CSS
  • How to improve form design and page presentation
  • How to document technical work clearly and professionally

Accessibility Considerations

Across all my pages I used semantic HTML elements such as <header>, <nav>, <main>, and <section> to improve page structure and accessibility. Images were given appropriate alt text, and form fields were clearly labelled where needed.

I also considered readability by using clear fonts, proper heading hierarchy, sufficient spacing, and responsive layouts to ensure the pages remain usable on different devices.

Compliance with JANET Regulations

All pages I created contain appropriate educational content and follow the project theme. I did not include harmful, misleading, or inappropriate material. My work was focused on original development and responsible use of content and assets.

I also ensured that my pages supported accessibility, usability, and good web design practice, which aligns with JANET compliance expectations for academic coursework websites.

My work supports responsible publishing through original content, accessible design, and appropriate use of external resources.
