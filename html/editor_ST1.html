Shared Template and Global CSS

HTML Structure

I created a shared HTML structure so that all group members could use the same layout across their assigned pages. The structure included a <header> for the site identity and navigation bar, a <main> area for page-specific content, and a <footer> with links to each student’s Page Editor and Validation page. This approach supported consistency and helped the group follow the required shared structure.

I used semantic HTML tags such as <header>, <nav>, <main>, <section>, and <footer> because they improve page structure, readability, and accessibility. This also supported easier collaboration because each group member could place their content inside the same reusable layout.

CSS Techniques

I created one external stylesheet, global.css, to control the shared appearance of the website. In this file I defined the body layout, typography, navigation styling, shared card design, buttons, image styles, footer formatting, form styles, tables, and responsive layout behaviour. I used a green-based palette to reflect the environmental theme of Life on Land and to keep the site visually connected across all pages.

I used:

  • Flexbox for aligning branding and navigation elements
  • CSS Grid for reusable card and tile layouts
  • Hover and focus states for links and buttons
  • Rounded corners and soft shadows to create a modern card-based design
  • Media queries to support smaller screens

Design and Implementation Rationale

I separated global and page-specific styling carefully. The shared stylesheet contains only reusable styles, while each major page uses embedded CSS for page-specific design. I chose this approach because the coursework brief requires the Global CSS to be shared and not to contain page-specific rules. This helped the group work more effectively and reduced the risk of style conflicts across pages.

Home Page

HTML Structure

The Home page was designed as the main entry point to the website. I used the shared structure and created a page layout with a hero section, a card-based mission section, an interactive tile area linking to the four student content pages, and a short action-focused section encouraging student participation.

I used <section> elements to divide the page into meaningful content areas and <article> elements for the three mission cards. I also used image elements with alternative text and unordered lists to highlight actions users could take.

CSS Techniques

In addition to the shared stylesheet, I added embedded CSS in the Home page to style the hero banner, image sizing, the interactive content gallery, and the page-specific highlight section. These embedded rules made it possible to customise the Home page without changing the shared stylesheet used by the rest of the team.

Design and Implementation Rationale

I designed the Home page to be visually welcoming and action-focused rather than descriptive. The content cards were written in original language to show why forests, wildlife, and ecosystem restoration matter. The interactive content tiles were added because the brief requires the Home page to provide visual links to all four student content pages.

Gallery Page

HTML Structure

The Gallery page contains four thumbnail cards, each representing a different Life on Land topic such as reforestation, wildlife protection, habitat restoration, and urban green spaces. Each card stores related information using custom data attributes, including the title, image path, image alternative text, description, and action tip.

JavaScript Techniques

I used JavaScript to implement the required gallery interactivity. Event listeners were added to each thumbnail so that when users click a card, a modal opens and displays the correct image, title, description, and action tip. I also added JavaScript to:

  • close the modal using a close button
  • close the modal when clicking outside it
  • close the modal using the Escape key
  • support keyboard opening using Enter or Space
  • change the colour scheme of the expanded view
  • change the font style of the expanded view

Design and Implementation Rationale

I used JavaScript rather than CSS-only interaction because the brief clearly states that the gallery interactivity must be implemented with JavaScript. I chose a modal approach because it lets users focus on one item at a time without leaving the page. The customisation controls only affect the expanded view, which directly satisfies the coursework requirement.

Content Page - Forest Conservation and Tree Planting

HTML Structure

My content page focuses on forest conservation and tree planting. I structured the page using one main heading and multiple subsections, including why forests matter, threats to forests, the value of tree planting, student actions, long-term benefits, and a conclusion. I also added an internal navigation menu near the top of the page to help users jump directly to different sections.

The page includes a Go to Top button implemented using an anchor link and fixed CSS positioning. This was added because the content page must provide visible internal navigation for longer content.

CSS Techniques

I used embedded CSS to style the hero section, section cards, two-column layouts, quote box, images, and fixed top button. I kept the styling consistent with the shared colour palette while making the page readable and visually organised.

Design and Implementation Rationale

I wrote original content instead of copying SDG material because the coursework asks students to create meaningful, original content relevant to the chosen SDG. I added internal anchors, image support, and structured sections so that the page would be easy to navigate and more accessible to users reading longer content.

Challenges and Lessons Learned

One of the main challenges I faced was creating a shared design that could be reused by all group members while still allowing each student to customise their own pages. To solve this, I separated global styles and page-specific styles carefully. The shared stylesheet contained only common elements such as navigation, cards, buttons, and layout rules, while each page used embedded CSS for unique visual behaviour.

Another challenge was implementing the Gallery page interaction in a way that clearly showed JavaScript use. I solved this by using data attributes and event listeners to open and update a modal dynamically, rather than relying on CSS-only techniques.

From this process, I learned:

  • how to organise reusable HTML and CSS for group projects
  • how to separate structure, style, and behaviour more clearly
  • how to use JavaScript event listeners and DOM updates in a practical way
  • how accessibility and consistency affect the quality of a website

Accessibility Considerations

Accessibility was considered throughout the pages I created. I used semantic HTML elements such as <header>, <nav>, <main>, and <section> to improve structure and help assistive technologies interpret the content more easily. I also added meaningful alt text to all images so that non-visual users can understand the purpose of visual content.

On the Gallery page, I added tabindex="0" to interactive thumbnail cards so that keyboard users can access them. I also supported Enter, Space, and Escape keys to improve keyboard interaction. In the shared CSS, I included visible focus states on links and form elements to help users understand which element is currently active.

I also considered readability by using:

  • high-contrast colours
  • clear headings and section structure
  • consistent spacing
  • responsive layouts for smaller screens

These choices improve usability and inclusivity for a wider range of users. The brief requires accessibility to be considered across implemented pages, and the compliance notes also emphasise accessible, inclusive web design.

Compliance with JANET Regulations

In the pages I created, I aimed to follow JANET-related good practice for responsible web publishing. My content is appropriate for an educational setting, focuses on a legitimate academic topic, and avoids harmful, misleading, or unlawful material. I also avoided including any malicious code or features that could misuse network resources.

I used original written content and planned to reference any external resources, such as images, properly. This supports ethical publishing and reduces the risk of copyright problems. I also avoided collecting unnecessary personal data. For example, the pages I developed do not request or process sensitive user information.

In addition, I considered accessibility as part of responsible publication. JANET compliance notes explain that web developers in education should use ethical content, respect copyright, avoid unnecessary data collection, and support accessibility and secure web practice.

My implementation supports responsible publishing through original content, accessible design, clear structure, appropriate educational content, and careful use of external materials.
