Reflection on Validation

Validation helped me identify and correct small markup issues during development, such as missing attribute values, link consistency, invalid semantic structures, and heading hierarchy problems that could affect standards compliance and accessibility. By checking each page carefully, I was able to improve code quality and ensure that the final pages followed valid HTML structure.

One important lesson I learned from the validation process is that even when a page looks correct in the browser, the underlying HTML can still contain errors or accessibility weaknesses. Using the validator helped me confirm that my pages were technically correct, easier to maintain, and ready for submission.
